
# MediTrack

## Home Page

This will be the landing page of our MediTrack webapp.

MediTrack is a web-based medication management application designed to help users track prescriptions, set refill alerts, and gain insights into their health trends. Developed by a team of Seneca Cloud Architecture and Administration students, this project offers a modern, responsive, and user-friendly landing page that serves as the gateway to a complete medication management experience.

## Overview

This repository contains the code for the home page / landing page of the MediTrack webapp. The landing page is built with HTML, CSS, and JavaScript and includes:

- A fixed navigation bar with smooth scrolling.
- A hero section with a call-to-action button.
- Sections for features, about us (including our story and team), contact, and authentication (login/sign-up).
- Responsive design with a mobile-friendly hamburger menu and animated elements.

## Features

- **Medication Management:** Track prescriptions, dosages, and refill dates.
- **Refill Alerts:** Receive timely notifications to ensure you never run out of medication.
- **Health Insights:** Monitor medication history and optimize your health plan.
- **User Authentication:** Seamlessly switch between login and sign-up forms.
- **Responsive Design:** Enjoy an optimal viewing experience on desktops, tablets, and mobile devices.

## Technology Stack

- **HTML5:** Provides the semantic structure of the landing page.
- **CSS:** Used for styling, animations, and responsive layouts.
- **JavaScript:** Handles interactive elements such as the mobile menu toggle and authentication form switching.

Open the Landing Page:

Use the navigation bar to explore different sections such as Features, About, Contact, and Authentication.

## Customization
HTML: Modify index.html to change the layout or add new sections.
CSS: Edit styles.css to adjust styling, colors, animations, and responsive behavior.
JavaScript: The interactive functionality is embedded in the HTML file. For more advanced functionality, consider separating the scripts into dedicated JavaScript files.
